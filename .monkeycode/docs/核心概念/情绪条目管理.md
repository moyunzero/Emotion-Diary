# 情绪条目管理

## 概述

情绪条目 (MoodEntry) 是应用的核心数据模型，记录用户的每一条情绪日记。

## 数据结构

```typescript
interface MoodEntry {
  id: string;                    // UUID 或时间戳字符串
  timestamp: number;              // 创建时间 (ms)
  moodLevel: MoodLevel;          // 情绪等级 1-5
  content: string;               // 日记正文
  deadline: string;               // 截止日期标签
  people: string[];              // 相关人物
  triggers: string[];            // 触发因素
  status: Status;                // active | resolved | burned
  resolvedAt?: number;            // 解决时间
  burnedAt?: number;              // 焚烧时间
  editHistory?: EditHistory[];    // 编辑历史
  audios?: AudioData[];           // 语音附件
  intensity?: 1 | 2 | 3 | 4 | 5; // 强度
  syncStatus?: SyncStatus;       // 同步状态
}
```

## 情绪等级

| 等级 | 枚举值 | 标签 | 图标 | 颜色 |
|------|--------|------|------|------|
| 1 | `MoodLevel.ANNOYED` | 有点委屈 | Droplet | #F59E0B |
| 2 | `MoodLevel.UPSET` | 心情低落 | Cloud | #F97316 |
| 3 | `MoodLevel.ANGRY` | 感到生气 | CloudRain | #EF4444 |
| 4 | `MoodLevel.FURIOUS` | 非常愤怒 | CloudLightning | #DC2626 |
| 5 | `MoodLevel.EXPLOSIVE` | 情绪爆发 | Zap | #991B1B |

## 条目状态

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| `active` | 活跃 | 创建时默认 |
| `processing` | 处理中 | 解决/焚烧操作中 |
| `resolved` | 已解决 | 用户标记为已解决 |
| `burned` | 已焚烧 | 用户执行焚烧操作 |

## 生命周期

```
[创建] ──► active ──┬──► resolved ──► (结束)
                    │
                    └──► burned ────► (结束)
```

## 操作

### 创建条目

```typescript
// 通过 store
await addEntry({
  moodLevel: MoodLevel.ANGRY,
  content: '今天被老板批评了',
  deadline: 'today',
  people: ['老板'],
  triggers: ['工作'],
  intensity: 4,
});
```

### 更新条目

```typescript
// 更新内容
updateEntry(id, { content: '新内容' });

// 自动记录编辑历史
```

### 解决条目

```typescript
resolveEntry(id);
// 设置 status = 'resolved'
// 设置 resolvedAt = Date.now()
```

### 焚烧条目

```typescript
burnEntry(id);
// 设置 status = 'burned'
// 设置 burnedAt = Date.now()
// 注意: 不会物理删除，只是标记
```

### 删除条目

```typescript
deleteEntry(id);
// 物理删除
// 同时删除关联的音频文件
```

## 编辑历史

每次更新条目时，自动记录编辑历史：

```typescript
interface EditHistory {
  editedAt: number;
  previousContent: string;
  previousMoodLevel: MoodLevel;
  previousDeadline: string;
  previousPeople: string[];
  previousTriggers: string[];
}
```

历史记录限制最新 10 条，超出后自动清理旧记录。

## 存储

- **本地**: AsyncStorage，键名 `entries_<userId>` 或 `entries_guest`
- **云端**: Supabase `entries` 表
