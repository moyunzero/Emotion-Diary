# 云端同步

## 概述

采用**离线优先 + 云端同步**策略。所有操作先作用于本地存储，网络恢复后自动同步到云端。

## 同步架构

```
┌─────────────────┐     ┌─────────────────┐
│   Local Store   │────►│  AsyncStorage   │
│   (Zustand)     │     │   (持久化)       │
└────────┬────────┘     └────────┬────────┘
         │                     │
         │  (debounced 300ms)  │
         ▼                     │
┌─────────────────┐             │
│  Sync Module    │             │
│  (防抖 + 互斥)   │             │
└────────┬────────┘             │
         │                      │
         ▼                      ▼
┌─────────────────────────────────────┐
│            Supabase                  │
│  ┌─────────┐  ┌──────────┐  ┌────┐ │
│  │  Auth   │  │ Database  │  │Storage│ │
│  └─────────┘  └──────────┘  └────┘ │
└─────────────────────────────────────┘
```

## 同步原则

1. **本地优先** - 所有操作先写本地，立即响应 UI
2. **最终一致** - 网络恢复后自动同步
3. **单向合并** - 以本地数据为准（当前版本）
4. **幂等操作** - 同步操作可重复执行

## 同步状态

```typescript
type SyncStatus = 'idle' | 'syncing' | 'pending' | 'error';

interface SyncModule {
  syncStatus: SyncStatus;
  syncToCloud: () => Promise<boolean>;
  syncFromCloud: () => Promise<boolean>;
  recoverFromCloud: () => Promise<boolean>;
}
```

## 同步流程

### 触发条件

1. **手动触发** - 用户下拉刷新
2. **定时触发** - 应用前台时定期同步
3. **变更触发** - 数据变更后防抖同步

### 同步互斥

使用 `isSyncingRef` 互斥锁防止竞态：

```typescript
let isSyncingRef = false;

const syncToCloud = async () => {
  if (isSyncingRef) {
    pendingSyncRef = true; // 标记待处理
    return false;
  }
  
  isSyncingRef = true;
  try {
    // 执行同步
  } finally {
    isSyncingRef = false;
  }
};
```

### 待处理请求合并

```typescript
let pendingSyncRef = false;

const processPendingSync = async () => {
  if (pendingSyncRef && !isSyncingRef) {
    pendingSyncRef = false;
    await useAppStore.getState().syncToCloud();
  }
};
```

## 数据同步

### 条目同步

```typescript
// 上传本地变更到云端
const uploadLocalEntries = async (entries: MoodEntry[], userId: string) => {
  for (const entry of entries) {
    if (entry.syncStatus === 'pending') {
      await supabase
        .from('entries')
        .upsert(convertToDbFormat(entry));
    }
  }
};

// 从云端下载最新数据
const downloadCloudEntries = async (userId: string) => {
  const { data } = await supabase
    .from('entries')
    .select('*')
    .eq('user_id', userId);
  return data;
};
```

### 音频同步

```typescript
// 上传待同步的音频
const uploadPendingAudios = async (audios: AudioData[], userId: string) => {
  for (const audio of audios.filter(a => a.syncStatus === 'pending')) {
    await uploadAudio(audio, userId);
  }
};
```

## firstEntryDate 同步

用户第一条记录的时间戳单独同步：

```typescript
// 上传
_syncFirstEntryDateToCloud: async () => {
  await supabase
    .from('users')
    .update({ first_entry_date: user.firstEntryDate })
    .eq('id', user.id);
};

// 下载
_syncFirstEntryDateFromCloud: async () => {
  const { data } = await supabase
    .from('users')
    .select('first_entry_date')
    .eq('id', user.id)
    .single();
  
  if (data?.first_entry_date) {
    // 合并取较早的
    updateFirstEntryDate(Math.min(localDate, cloudDate));
  }
};
```

## Supabase 配置

### 环境变量

```bash
EXPO_PUBLIC_SUPABASE_URL=<your-supabase-url>
EXPO_PUBLIC_SUPABASE_ANON_KEY=<your-anon-key>
```

### 离线降级

如果未配置 Supabase，应用以纯离线模式运行：

```typescript
// lib/supabase.ts
const supabaseUrl = process.env.EXPO_PUBLIC_SUPABASE_URL || '';
const supabaseAnonKey = process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY || '';

export const isSupabaseConfigured = (): boolean => {
  return !!(supabaseUrl && supabaseAnonKey);
};
```

## 错误处理

| 错误类型 | 处理策略 |
|----------|----------|
| 网络错误 | 标记 `syncStatus: 'pending'`，网络恢复后重试 |
| 认证错误 | 清除会话，引导重新登录 |
| 服务器错误 | 记录日志，重试 3 次 |
