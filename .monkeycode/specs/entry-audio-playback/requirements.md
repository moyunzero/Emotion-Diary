# EntryCard 语音播放功能增强

## 1. 功能概述

在 `EntryCard` 展开状态下，为语音附件添加播放功能，允许用户直接在卡片内回顾语音内容。

## 2. 用户需求

**用户故事：** AS 用户，我想要在查看历史情绪记录时能够直接播放语音附件，以便回顾当时录制的语音内容。

## 3. UI/UX 设计

### 3.1 缩略状态（未展开）
- 保持现状：仅显示 `{N}条语音` 静态标签

### 3.2 展开状态
- 在标签区域下方新增"语音播放区"
- 每条语音显示：
  - 麦克风图标
  - 录音名称（默认：`录制于 {时间}`）
  - 播放/暂停按钮
- 点击播放按钮：
  - 按钮变为暂停图标
  - 当前项高亮背景
- 播放完成：自动停止，按钮恢复

### 3.3 播放控制
- 点击播放：开始播放
- 再次点击：暂停
- 播放完成：自动停止

## 4. 技术方案

### 4.1 状态管理
- `playingAudioId: string | null` - 当前播放的音频ID
- `playbackPosition: number` - 播放进度（秒）

### 4.2 音频 URL 解析
- 优先使用 `audio.localUri`
- 本地不存在时使用 `audio.remoteUrl`

### 4.3 组件隔离
- 每个 EntryCard 独立管理播放状态
- 不同卡片互不影响

## 5. 验收标准

1. 展开的 EntryCard 显示语音播放列表
2. 点击播放按钮可正常播放语音
3. 播放时显示暂停按钮和播放指示
4. 播放完成自动停止
5. 缩略状态保持原样
