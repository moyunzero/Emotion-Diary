# 项目概览规范

## 项目目标和愿景

情绪日记（Emotion Diary）是一款专注于情绪管理的治愈系应用。通过独特的「情绪气象站」和「心灵花园」概念，帮助用户记录、理解和管理自己的情绪，让每一次情绪的记录和解决都成为照料心灵花园的过程。

### 核心价值

- **治愈系体验**：温暖舒适的视觉设计和交互体验
- **情绪管理**：帮助用户理解和处理负面情绪
- **关系健康**：通过情绪记录改善人际关系
- **隐私保护**：离线优先，本地存储保护用户隐私

## 技术栈

依赖版本、Native 模块与 CI 的**权威锚点**见 [`engineering-system.md`](./engineering-system.md) §6（勿在本文件维护版本号表）。

**非技术读者一句话**：跨 iOS / Android / Web 的情绪记录应用；基于 Expo 与 React Native；数据默认存本地、可选云端同步；可选 Groq AI 辅助分析。图标与动效以 Lucide、Reanimated、Skia 等为主，细节以 `engineering-system.md` 为准。

## 架构概览

分层边界、同步路径与代码锚点见 **[`engineering-system.md`](./engineering-system.md)**；**工程 vs 领域文档分工**见 **`engineering-system.md` / `engineering-quality.md` 文首 SDD 说明**。以下为产品向摘要。

### 数据流

1. **用户交互** → UI 组件触发操作
2. **状态更新** → Zustand Store 更新状态
3. **数据持久化** → 保存到 AsyncStorage 或 Supabase
4. **数据同步** → 云端数据同步（如果配置）

### 关键设计原则

- **离线优先**：所有数据首先存储在本地，云端同步为可选功能
- **组件化**：UI 组件高度模块化，便于复用和维护
- **类型安全**：使用 TypeScript 确保类型安全
- **响应式设计**：适配不同屏幕尺寸和设备

## 核心功能模块

### 1. 情绪气象站（Weather Station）
- 创新的天气隐喻可视化关系健康状态
- 用天气图标（雨滴、云朵、雷电等）表达情绪强度
- 实时显示当前"关系天气"和情绪指数

### 2. 智能记录（Smart Recording）
- 5级情绪强度记录（从"有点委屈"到"情绪爆发"）
- 天气主题图标系统
- 多维度标签系统（人物标签和情绪触发器标签）
- 灵活的期限设置
- 自动草稿保存
- 编辑历史记录

### 3. 心灵花园（Insights）
- 本周情绪天气可视化
- 治愈进度环形图（情绪解决率）
- 关系花盆可视化（关系健康度）
- 情绪触发洞察（Top 3 情绪触发器）
- 底部鼓励语

### 4. 气话焚烧（Burn Feature）
- 治愈系情绪释放功能
- Skia 燃烧动画效果
- 让负面情绪随火焰消散

### 5. AI 智能助手
- 情绪预测：基于历史数据预测未来7天情绪走势
- 情绪播客：AI 生成个性化情绪疗愈播客内容
- 情绪处方：针对触发器提供个性化建议和应对策略
- 智能分析：深度分析情绪周期和触发因素

### 6. 数据同步
- 离线优先：本地存储保护用户隐私
- 云端备份：可选 Supabase 云端同步
- 智能数据迁移：支持访客数据与登录用户数据无缝切换

## 设计规范

### 配色方案
- **治愈系配色**：粉绿渐变主题，温暖舒适
- **情绪等级颜色**：黄色（轻微）→ 橙色（低落）→ 红色（生气）→ 深红（爆发）

### 图标系统
- **统一使用 Lucide 图标库**：避免 emoji 兼容性问题
- **天气主题图标**：Droplet、Cloud、CloudRain、CloudLightning、Zap

### 交互设计
- **流畅动画**：React Native Reanimated 驱动的微交互
- **响应式设计**：适配各种屏幕尺寸
- **正向激励**：强调成长和治愈，而非问题和冲突
