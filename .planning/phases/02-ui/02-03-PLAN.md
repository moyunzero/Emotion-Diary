# Plan 02-03：expo-media-library、权限与首次保存隐私提示

## Goal

安装并配置 **`expo-media-library`**：在 **`保存到相册`** 流程中 **请求写入权限**、将 **02-02** 产出的 **临时 PNG URI** 写入 **系统相册**；**首次**保存前展示 **`Alert` 或 Modal**（图含情绪信息、保存后相册可见），可选 **「不再提示」** 写入 **`AsyncStorage`** 键 **`review_export_privacy_ack_v1`**。**不** 使用 `expo-sharing`。

## must_haves

1. `npx expo install expo-media-library` 后 **`package.json`** 含 **`expo-media-library`** 依赖。  
2. iOS **`NSPhotoLibraryAddUsageDescription`**（或 Expo 文档要求的相册写入键）在 **`app.json` → `expo.ios.infoPlist`** 与预构建 **`ios/app/Info.plist`** 中一致、中文说明含 **「保存回顾图到相册」** 语义。  
3. Android：**`READ_MEDIA_IMAGES` / `READ_EXTERNAL_STORAGE`** 等以 **Expo media-library 文档当前推荐** 为准，并在 **`AndroidManifest.xml`** / **`strings.xml`** 有对应说明（若插件注入则只改 `app.json` 插件配置亦可，须能通过 `verify:permissions` 或文档自述）。  
4. 保存成功后可 **`Alert.alert('已保存到相册')`** 或 Toast；失败 **`Alert` 显示 `error.message`**。

## Verification

- [ ] `grep "expo-media-library" package.json` 命中  
- [ ] `grep "review_export_privacy_ack_v1" components/ReviewExport/ReviewExportScreen.tsx` 或 `store`/工具文件命中  
- [ ] `grep "MediaLibrary" components/ReviewExport/ReviewExportScreen.tsx` 命中  
- [ ] `grep "expo-sharing" package.json` **无** 新增（不应安装）
