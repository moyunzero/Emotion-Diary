# Plan 02-01：预设时间范围、路由、回顾画布与洞察入口

## Goal

实现 **`/review-export`** 全屏页：**本周/本月/上周/上月** 预设（默认 **本月**）、与 `2-UI-SPEC.md` 一致的 **可截图画布**（页眉、解决率+环比+笔数、趋势占位或简易图、Top3 天气、Top3 触发器、**非 AI 占位总结**），并从洞察页 **「生成情绪回顾图」** 进入。本计划 **不** 实现 `captureRef` 成片与 **不** 实现相册写入（属 02-02 / 02-03）。

## must_haves

1. `router.push('/review-export')` 可达；返回可用 `router.back()`。  
2. 切换预设时 `startMs/endMs` 与 **环比上一同期** 边界正确（月对月、周对周，与 `reviewStatsTimeRange` 辅助函数一致）。  
3. 画布展示数字与 Phase 1 纯函数一致：`compareResolutionToPreviousPeriod`、`getTopThreeWeatherBucketsByDays`、`getTopTriggersWithAdvice`、`calculateDaysAsOf`、`getMonthlyResolutionRateSeries`（趋势）。  
4. 主按钮文案为 **`保存到相册`**；`onPress` 可先 **`Alert.alert`** 占位 **「导出功能将在下一步接入」** 或 **空函数**，**不得** 出现 `Share`、`expo-sharing`。

<action>
在 **`utils/reviewStatsTimeRange.ts`** 中新增（命名可微调，须导出并被回顾屏引用）：

1. **类型** `ReviewExportPreset` = 字面量联合 **`'this_week' | 'last_week' | 'this_month' | 'last_month'`**。  
2. **函数** `getReviewExportPeriods(now: Date, preset: ReviewExportPreset): { current: { startMs: number; endMs: number }; previous: { startMs: number; endMs: number } }`：  
   - **`this_month`**：`current` = `getCalendarMonthRange(now.getFullYear(), now.getMonth())`；`previous` = `getPreviousCalendarMonthRangeBefore(current.startMs)`。  
   - **`last_month`**：`current` = `getPreviousCalendarMonthRangeBefore(getCalendarMonthRange(now.getFullYear(), now.getMonth()).startMs)`；`previous` = `getPreviousCalendarMonthRangeBefore(current.startMs)`。  
   - **`this_week`**：`current` = `getMondayWeekRangeContaining(now)`；`previous` = `getPreviousWeekRange(current.startMs, current.endMs)`，若返回 `null` 则与调用方约定 fallback（文档注释说明）。  
   - **`last_week`**：先取 `thisWeek = getMondayWeekRangeContaining(now)`，再 `lastWeek = getPreviousWeekRange(thisWeek.startMs, thisWeek.endMs)` 作为 **`current`**；`previous` = 再对 `lastWeek` 调一次 `getPreviousWeekRange`（若 `null` 注释说明）。  

3. 文件底部 **export** 上述符号。
</action>
