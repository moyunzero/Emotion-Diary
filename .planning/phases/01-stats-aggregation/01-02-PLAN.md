# Plan 01-02：周期内记录数、和解数、解决率、环比

## Goal

实现 **`utils/reviewStats.ts`**：在给定 `startMs/endMs` 下从 `MoodEntry[]` 计算 **total、resolved、rate**，以及 **相对上一同期** 的解决率与差值（`1-CONTEXT` D-04～D-10）。

## must_haves

1. 周期内 `timestamp ∈ [startMs, endMs]` 的过滤与计数与 CONTEXT 一致。  
2. `total === 0` 时 rate 与环比结构有明确类型（如 `null` 或 `{ currentRate: null }`，在导出层处理）。  
3. 单测覆盖典型：全空、仅 resolved、两期对比。

## Verification

- [ ] `grep "filterEntriesByTimestampRange" utils/reviewStats.ts` 或等价实现存在  
- [ ] `yarn test:unit --testPathPattern=reviewStats` 退出码 0

<action>
新建 **`utils/reviewStats.ts`**，导出：

1. **类型**（可内联于本文件）：  
   - `ResolutionPeriodStats`：`{ total: number; resolved: number; resolutionRate: number | null }`  
     - `resolutionRate` = `total === 0 ? null : resolved / total`  
   - `ResolutionCompare`：`{ current: ResolutionPeriodStats; previous: ResolutionPeriodStats; deltaRate: number | null }`  
     - `deltaRate` = `current.resolutionRate !== null && previous.resolutionRate !== null ? current.resolutionRate - previous.resolutionRate : null`（小数差，如 0.12 表示 +12 个百分点若 UI 乘 100）

2. **`filterEntriesInRange(entries: MoodEntry[], startMs: number, endMs: number): MoodEntry[]`**  
   - 条件：`e.timestamp >= startMs && e.timestamp <= endMs`

3. **`getResolutionPeriodStats(entries: MoodEntry[], startMs: number, endMs: number): ResolutionPeriodStats`**  
   - `total` = 过滤后条数（**所有**状态，含 `BURNED`）。  
   - `resolved` = 过滤后 `status === Status.RESOLVED` 条数。

4. **`compareResolutionToPreviousPeriod(entries, currentStart, currentEnd, previousStart, previousEnd): ResolutionCompare`**  
   - 调用两次 `getResolutionPeriodStats` 并计算 `deltaRate`。

文件顶部注释：**与 `components/Insights/index.tsx` 全量统计不同**，本模块仅服务「闭区间」回顾导出。
</action>
