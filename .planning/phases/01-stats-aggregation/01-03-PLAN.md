# Plan 01-03：Top 天气（按天）、Top 触发器、月度解决率序列

## Goal

在 **01-02** 的过滤与解决率基础之上，实现 **D-14～D-19** 与 DEFERRED 中的 **按月解决率序列**（供 Phase 4 图表消费）。

## must_haves

1. 每日 `max(moodLevel)` → 四类 `sunny|cloudy|rainy|stormy` → 按 **出现天数** Top 3。  
2. 触发器计数与 `TriggerInsight` 一致；园艺文案从 `TRIGGER_ADVICE` 取。  
3. 可选：`getMonthlyResolutionRateSeries(entries, count)` 返回最近 `count` 个自然月的 `{ year, monthIndex0, rate }[]`。

## Verification

- [ ] `grep "moodLevelToExportWeatherBucket" utils/reviewStatsWeather.ts` 或等价映射函数存在  
- [ ] `grep "TRIGGER_ADVICE" utils/reviewStats.ts` 或 `reviewStatsTriggers.ts`  
- [ ] 相关 `yarn test:unit` 子集通过

3. **`getTopThreeWeatherBucketsByDays(...)`**  
   - 返回 `{ bucket, days, labelZh }[]` 长度 ≤3，`labelZh` 建议固定：**晴朗 / 多云 / 有雨 / 暴风雨**（与产品示例一致）。

在 `utils/reviewStats.ts` **或**本文件末尾导出聚合入口 **`getReviewWeatherSummary(entries, startMs, endMs)`**（任选其一文件，避免循环依赖）。
