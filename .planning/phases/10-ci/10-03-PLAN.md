  <name>Task 1: 添加分层 CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <read_first>
    package.json
    .planning/phases/10-ci/10-RESEARCH.md
    scripts/verify-governance-smoke.js
  </read_first>
  <action>
新建 `.github/workflows/ci.yml`，YAML 内容满足以下 **硬约束**（可在此基础上增加 `workflow_dispatch` 或 `concurrency`，但不得删以下行为）：

1. 文件顶：`name: CI`
2. 触发器：
   ```yaml
   on:
     pull_request:
     push:
       branches: [main]
   ```
3. Job `pr-gate`：
   - `runs-on: ubuntu-latest`
   - `if: github.event_name == 'pull_request'`
   - steps 顺序：`actions/checkout@v4` → `actions/setup-node@v4` with `node-version: '20'` and `cache: 'yarn'` → `run: yarn install --frozen-lockfile` → `run: yarn lint` → `run: yarn test:ci`
4. Job `governance`：
   - `runs-on: ubuntu-latest`
   - `if: github.event_name == 'push'`
   - 相同 checkout / setup-node / `yarn install --frozen-lockfile`
   - `run: yarn lint`
   - `run: yarn test:ci`
   - `run: yarn verify:governance`
   - `run: node scripts/verify-governance-smoke.js`
5. **不要**在 `pr-gate` 中加入 `verify:governance` 或 `verify-governance-smoke`（保持 PR 快路径；per 10-RESEARCH 主建议）。
  </action>
  <verify>
    <automated>test -f .github/workflows/ci.yml && rg -q "pr-gate:" .github/workflows/ci.yml && rg -q "governance:" .github/workflows/ci.yml && test "$(rg -c 'yarn verify:governance' .github/workflows/ci.yml | tr -d '[:space:]')" = "1"</automated>
  </verify>
  <acceptance_criteria>
    - 文件 `.github/workflows/ci.yml` 存在
    - 该文件含字符串 `if: github.event_name == 'pull_request'` 与子串 `pr-gate`
    - 该文件含字符串 `if: github.event_name == 'push'` 与子串 `governance`
    - 命令 `rg -c 'yarn verify:governance' .github/workflows/ci.yml` 的输出去空白后等于 `1`（仅 governance job 调用一次）
    - 该文件含字面量 `node scripts/verify-governance-smoke.js`
    - 该文件至少两处含 `yarn install --frozen-lockfile`，且至少一处含 `cache: 'yarn'`
  </acceptance_criteria>
  <done>分层 CI workflow 已落库</done>
