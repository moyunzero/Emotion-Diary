# App Review 回复模板（4.3(a)）

Hello App Review Team,

Thank you for your feedback regarding Guideline 4.3(a).
This build focuses on clear differentiation in branding, core user flow, and review-verifiable behavior.

## Changes in this build

1. Unified product identity across metadata and in-app wording as **焚语 / Fenyu**.
2. Removed template-style elements (generic placeholders and non-product random avatar behavior).
3. Clarified the product's core narrative as:
   - Emotion logging with the Mood Weather flow
   - Weekly/monthly review export, saved to Photos for personal reflection
4. Kept AI output as a secondary assistant feature with fallback text, not a primary app claim.

## Verification paths

1. **Core flow A (record to archive):**
   Record emotion -> Open entry -> Burn interaction -> Insights -> Emotion Release Archive
2. **Core flow B (trigger action loop):**
   Insights -> Trigger Insight -> Action -> Revisit
3. **Core flow C (review export boundary):**
   Review Export -> Generate review image -> Save to Photos
   (No built-in third-party sharing entry inside the app)

## Distinct branding and functionality statement

- The app uses a specific emotional journaling identity and language style (**焚语 / Fenyu**), not generic template branding.
- The primary user value is personal emotional tracking and review export continuity.
- The release/archive and trigger-action-revisit loop are app-specific behavioral flows.

## Compliance boundary statement

- The app does not provide medical diagnosis, therapy, or treatment claims.
- Exported review content is user-generated emotional content and is saved to the user's own photo library by explicit user action.
