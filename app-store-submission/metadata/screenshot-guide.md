# 焚语 App Store 截图指南（最小可过审包）

目标：用 5 张截图完成「记录情绪 -> 回顾导出 -> 相册可见与隐私提醒」主链路，并保留产品差异化能力。

## 1. 情绪记录入口（主价值起点）

- 目标信息点：用户可以快速记录当下情绪，不是通用记事本。
- 来源页面：首页/情绪气象站（含天气强度入口）。
- 必须出现的真实 UI 元素：情绪气象站标题、天气情绪图标、记录按钮或记录入口。
- 建议标题（短句，8-14 字）：`今天的情绪，马上记下`
- 禁止项：placeholder 文案、测试账号昵称、空白卡片状态。

## 2. 周/月回顾页面（主价值延展）

- 目标信息点：可按周或按月查看情绪变化，形成连续回顾。
- 来源页面：回顾导出页（时间范围选择与统计区域）。
- 必须出现的真实 UI 元素：周/月范围、陪伴天数、记录笔数或和解笔数。
- 建议标题（短句，8-14 字）：`每周每月，看见变化`
- 禁止项：与当前时间不一致的假日期、不可读小字、演示水印。

## 3. 导出结果预览（导出到相册）

- 目标信息点：回顾结果可导出为图片并保存到系统相册。
- 来源页面：回顾导出预览区域/导出前后状态页。
- 必须出现的真实 UI 元素：导出按钮、导出结果预览、保存成功反馈（如 toast/提示文案）。
- 建议标题（短句，8-14 字）：`回顾图，一键存入相册`
- 禁止项：第三方分享按钮占主位、失败弹窗停留、调试开关。

## 4. 隐私提示与边界（审核关键信息）

- 目标信息点：情绪内容有隐私属性，导出后需注意相册可见范围；应用内不内置第三方分享。
- 来源页面：导出前隐私提醒弹窗或设置中的隐私说明区域。
- 必须出现的真实 UI 元素：隐私提醒文案、用户确认动作（继续/取消）、与导出相关说明。
- 建议标题（短句，8-14 字）：`保存前，先确认隐私`
- 禁止项：治疗/诊断措辞、夸张承诺、模糊到看不清的提示内容。

## 5. 情绪释放档案与行动闭环（差异化支撑）

- 目标信息点：不仅记录，还可在释放档案中回看并形成行动回访闭环。
- 来源页面：情绪释放档案、触发器洞察或行动闭环相关页面。
- 必须出现的真实 UI 元素：释放档案条目、触发器信息、行动或回访标记。
- 建议标题（短句，8-14 字）：`释放之后，也能回看成长`
- 禁止项：空数据列表、开发占位图、与主叙事无关的次要功能抢主视觉。

## 通用制作规范

- 使用真实数据，不用 placeholder、测试数据、测试账号。
- 品牌统一使用 `焚语 / Fenyu`。
- 每张截图只表达一个核心差异点，避免信息拥挤。
- 截图顺序需严格按 1 -> 5 递进，映射文案主叙事。
